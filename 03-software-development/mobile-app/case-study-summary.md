# MealMoxie — Case Study Summary

## The Brief
MealMoxie needed a mobile app MVP for corporate meal benefits — real-time ordering, daily credits, corporate dashboard — in 12 weeks with $2.1M seed funding.

## Approach
- React Native (70% code sharing iOS + Android)
- Validation sprint with 24 employee interviews
- 3 corporate beta partners (150 users) before public launch
- Offline-first architecture for corporate WiFi reliability
- Stripe Connect for corporate billing

## Results
- **Timeline:** 12 weeks, on budget
- **Users:** 0 → 8,900 in 6 months
- **GMV (Month 6):** $890K
- **App Store rating:** 4.7
- **Corporate retention:** 97%
- **Orders:** 31K/month at Month 6
- **Corporate partners:** 3 → 34 in 6 months

## Tech Stack
React Native (Expo), TypeScript, Node.js, PostgreSQL (Railway), Redis, Stripe Connect, Algolia, Google Places, Firebase, Next.js

## Key Differentiator
Offline-first architecture. Corporate WiFi is unreliable — orders queue locally and sync when connected. Key differentiator vs DoorDash for Business.
