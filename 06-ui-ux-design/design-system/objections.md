# Stratos — Objection Handling

## "Our teams are already productive with current components"

**Acknowledge:** "Current components work, absolutely."

**Pivot:** "But 'working' isn't the same as 'efficient.' Every sprint, engineers rebuild UI that should already exist. The 47 button variants take time to maintain. A design system means never rebuilding basic UI again."

---

## "A design system will slow us down during migration"

**Acknowledge:** "Migration takes time—that's real."

**Pivot:** "We built the system for incremental adoption. Teams can migrate feature-by-feature, not all at once. We phase rollout over 12 weeks. And after migration, teams go 38% faster."

---

## "Our engineers will fight losing control of their components"

**Acknowledge:** "Engineers are protective of their work."

**Pivot:** "We involved engineers from day one. They helped design the component APIs. The system isn't imposed—it's co-created. Plus, teams can still build custom components for genuine needs."

---

## "We don't have resources to migrate everything"

**Acknowledge:** "Migration is an investment."

**Pivot:** "We provide migration support: playbooks, training, office hours. We also prioritize components by usage—highest-impact components migrate first. You don't need to migrate everything to see benefits."

---

## "A design system will make everything look the same"

**Acknowledge:** "Uniformity is a legitimate concern."

**Pivot:** "The system provides foundations, not limitations. 85 components can be composed in countless ways. Enterprise theming allows brand customization. The system enables creativity, not replaces it."

---

## "How do we handle custom enterprise requirements?"

**Acknowledge:** "Enterprise customers have unique needs."

**Pivot:** "We built a theming system for white-labeling. Enterprise can customize colors, fonts, and some behaviors without breaking components. GlobalTech used this to match their brand while using standard components."

---

## "Our tech stack is mixed—some React, some Vue"

**Acknowledge:** "Mixed stacks are common."

**Pivot:** "We built the system with multi-framework support. React primary, with Vue wrappers. HTML components for emails and docs. The design tokens are framework-agnostic."

---

## "Who owns the design system after launch?"

**Acknowledge:** "Ownership is critical for sustainability."

**Pivot:** "We establish governance during the project: system champions in each team, clear contribution processes, quarterly roadmap reviews. The design system becomes part of the product organization, not a side project."
